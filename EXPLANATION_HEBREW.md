# הסבר על הפרויקט - Cost Manager

## מה עושה האפליקציה?

האפליקציה היא **מנהל הוצאות** שמאפשרת:
- להוסיף הוצאות (סכום, מטבע, קטגוריה, תיאור)
- לראות דוח חודשי של ההוצאות
- לראות גרף עוגה של הוצאות לפי קטגוריות
- לראות גרף עמודות של הוצאות לאורך השנה
- להמיר בין מטבעות (USD, ILS, GBP, EURO)

## הסבר על הטכנולוגיות

### IndexedDB
מסד נתונים שנמצא **בתוך הדפדפן**. כל המידע נשמר מקומית במחשב של המשתמש.
- לא צריך שרת
- המידע נשאר גם אחרי סגירת הדפדפן
- כל משתמש רואה רק את המידע שלו

### React
ספרייה לבניית ממשקי משתמש. במקום לכתוב HTML רגיל, כותבים "קומפוננטות" (חלקים קטנים שמרכיבים את הדף).

### MUI (Material-UI)
ספריית עיצוב מוכנה. במקום לכתוב CSS מאפס, משתמשים בכפתורים, טבלאות, ושדות קלט מעוצבים מראש.

### Chart.js
ספרייה ליצירת גרפים (עוגה, עמודות, קווים וכו').

### Vite
כלי שמריץ את הפרויקט בזמן פיתוח ובונה אותו להעלאה לשרת.

### Fetch API
דרך להביא מידע משרת. בפרויקט שלנו משתמשים בזה כדי להביא שערי חליפין.

## שערי חליפין - איך זה עובד?

1. יש קובץ `rates.json` שנראה ככה:
```json
{
  "USD": 1,
  "GBP": 0.79,
  "EURO": 0.92,
  "ILS": 3.65
}
```

2. הקובץ הזה חייב להיות על **שרת** (לא מקומי על המחשב)

3. האפליקציה מביאה את הקובץ הזה באמצעות Fetch API

4. לפי השערים, האפליקציה ממירה בין מטבעות

### איפה הקובץ נמצא עכשיו?
```
https://raw.githubusercontent.com/RoniBaranes/client-side-new/main/public/rates.json
```

### מה המשמעות של השערים?
- USD 1 = USD 1 (בסיס)
- GBP 0.79 = USD 1 (ליש"ט אחד שווה 1.27 דולר)
- EURO 0.92 = USD 1 (יורו אחד שווה 1.09 דולר)
- ILS 3.65 = USD 1 (3.65 שקלים שווים דולר אחד)

## שאלות נפוצות

### למה יש שני קבצי idb.js?
הדרישות מבקשות **שתי גרסאות**:
1. `public/idb.js` - גרסה "וניל" (JavaScript רגיל) לבדיקה
2. `src/db/idb-react.js` - גרסה עם מודולים לשימוש ב-React

### למה צריך שרת לשערי חליפין?
הדרישות אומרות שחייבים להביא את השערים מ"server-side" באמצעות Fetch API. לכן העלינו את הקובץ ל-GitHub.

### מה זה Render.com?
שירות אירוח אתרים בחינם. שם מעלים את האפליקציה כדי שתהיה נגישה באינטרנט.

### למה יש תיקיית components ריקה?
זו תיקייה שנוצרה ולא נמחקה. אפשר למחוק אותה.
